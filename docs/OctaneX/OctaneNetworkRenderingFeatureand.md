OctaneRender's network rendering feature is done from the program itself - it does not control the network rendering implementations of other 3D applications (e.g., tap into TeamRender and/or NetRender), and it communicates just with the GPUs. Deploying another network render other than OctaneRender's network render requires the peers/clients at the other end to translate OctaneRender® properties, so this means the OctaneRender plugin is necessary at the peer/client side despite the host 3D application's network schema being used.
